# Mind-Match
SCC-based community detection in directed interest graphs.

Course: TCS-409 Design and Analysis of Algorithms
Team: DataStellar (DAA-IV-T151)
Members: Atishay Jain, Sarthak Kathait, Suhani Dangwal, Vinayak Singh

Mind-Match builds a directed graph of users based on Jaccard similarity of interests, detects communities as strongly connected components using Kosaraju and Tarjan, benchmarks both algorithms, and exports JSON for visualization. The repo also includes a real-time Node.js + Socket.io dashboard that drives the C++ engine.

## Highlights

- SCC detection with Kosaraju and Tarjan
- Validation that both algorithms produce identical partitions
- JSON export with metrics, timings, users, edges, and communities
- Real-time dashboard with graph visualization, analytics, chat, and simulation tools
- Legacy static UI for quick offline visualization

## Quick start (C++ CLI)

Prerequisites:
- g++ with C++17 support
- make

Build:
```bash
make
```

Run with sample data:
```bash
./mindmatch data/sample.csv 0.30
```

Generate synthetic data:
```bash
./mindmatch --generate 100
./mindmatch data/generated.csv 0.25
```

Export JSON for the UI:
```bash
./mindmatch --json data/sample.csv 0.30 > ui/result.json
```

Windows note:
If make is not available, compile with g++ using the commands in PROJECT_PROGRESS.md.

## Web UI

### Real-time dashboard (Node.js + Socket.io)

Prerequisites:
- Node.js 18+

Run the server:
```bash
cd server
npm install
npm start
```

Open http://localhost:3000 in your browser. The server seeds users from data/sample.csv on first run and invokes the C++ engine to compute communities.

Dashboard features:
- Live community graph (D3.js) with focus mode and tooltips
- Real-time SCC recomputation on data changes
- Community chat, join requests, and notifications
- Analytics panel with algorithm timings and distributions
- Simulation tools (bulk add/remove, live evolution)
- Interest intelligence and cross-community chat

### Legacy static visualization

If you only need a static visualization, open ui/index_legacy.html and upload a JSON file generated by the CLI:

```bash
./mindmatch --json data/sample.csv 0.30 > ui/result.json
```

## Algorithms

- Kosaraju and Tarjan both run in O(V + E)
- Kosaraju uses two DFS passes and a reversed graph
- Tarjan uses one DFS pass with low-link tracking

## JSON output (high level)

The CLI exports a single JSON object with these top-level keys:

```
meta
metrics
timings_microseconds
users
edges_list
communities
```

Notes:
- meta.validation_passed indicates whether Kosaraju and Tarjan match
- timings_microseconds includes per-algorithm timings and speedup ratio
